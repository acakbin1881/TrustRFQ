Canonical golden-vector capture

Runs fixed orders through canonical.js — loaded from the same esm.sh URLs the live desk uses — and records the resulting fill arguments as base64 XDR. The React port's unit test must reproduce these bytes exactly; a substituted Buffer or a reordered argument then fails a test instead of reverting a real settlement on-chain.

Regenerate with node tools/capture-server.mjs (drives this page headlessly and writes fixtures/canonical-args.json), or just open this page and copy the JSON.
